泛型编程->模板与STL技术

一、模板

1. 模板的概念

  1. 建立通用模具，提高代码复用性

2. 函数模板

1. 作用

建立一个通用函数，其函数 返回值类型和形参类型可以不具体确定，用一个虚拟类型来代表

2. 定义语法：

template<typename T>函数

  1. template->声明创建模板
  2. typename->表明其后面的符号是一种数据类型，可以用class代替
  3. T->通用的数据类型，名称可以替换，通常为大写字母

3. 使用语法

  1. 自动类型推导
  2. 显式指定类型

4. 注意事项

  1. 自动类型推导必须推导出一致的数据类型T
    1. 如上述函数不能输入int a和double b;
  2. 模板必须要能确定出T的数据类型
    1. 如函数中不使用参数，则类型T无法确定

5. 函数模板案例

  1. 对不同类型数据进行排序
  2. 从大到小，使用选择排序
  3. 分别利用char数组和int数组实现

6. 普通函数与函数模板的区别

  1. 普通函数调用时可以发生自动类型转换（隐式类型转换）
  2. 函数模板 使用自动类型推导，不可以隐式类型转换
  3. 使用显式指定类型，可以隐式转换

7. 普通函数与模板函数的调用规则

  1. 函数模板和普通函数都可以实现，优先调用普通函数
  2. 可以通过空模板参数列表强制调用函数模板
  3. 函数模板也可以函数重载
  4. 如果函数模板可以产生更好的匹配，优先调用函数模板

    既然提供了函数模板，就不要再定义普通函数而出现二义性

8. 模板的局限性

  • 特定数据类型需要使用具体方式进行特殊实现
  • 利用具体化模板，可以解决自定义类型的通用化
  • 学习模板并非为了写模板，而是在STL中能够运用系统提供的模板

3. 类模板

1. 类模板语法

  1. 作用：建立一个通用类，类中的成员数据类型使用虚拟类型
  2. 语法：

2. 类模板与函数模板区别

  1. 类模板没有自动类型推导的使用方法
  2. 类模板在模板参数列表中可以有默认参数

3. 类模板中成员函数的创建时机

  1. 普通类中的成员函数一开始就可以创建
  2. 类模板中的成员函数在调用时才创建

4. 类模板对象做函数参数

三种传入方式：

  1. 指定传入类型->直接显示对象的数据类型(常用)
  2. 参数模板化->将对象中的参数变为模板进行传递
  3. 整个类模板化->将这个对象类型 模板化进行传递

5. 类模板与继承

  1. 当子类继承的父类是一个类模板时，子类在声明时要指定出父类中T的类型
  2. 若不指定，编译器无法给子类分配内存
  3. 若想灵活指定父类中T的类型，子类也需变为类模板

二、STL

1. STL基础

1. 概念

  1. Standard Template Library（标准模板库）:基于模板实现的C++基础库
  2. STL从广义上分为：the Containers STL容器，the Iterators STL迭代器，the Algorithms STL算法
  3. 容器和算法之间通过迭代器进行无缝连接
  4. STL几乎所有的代码都采用了模板类或者模板函数

2. STL六大组件

  1. 容器：各种数据结构，如vector、list、deque、set、map等，用来存放数据
    1. 序列式容器：强调值的排序，容器中的每个元素均有固定位置
    2. 关联式容器：二叉树结构，各元素之间没有严格的物理上的顺序关系
  2. 算法：各种常用的算法，如sort、find、copy、for_each等
    1. 质变算法：：会改变容器的内容，例如拷贝、替换、删除
    2. 非质变算法：不会改变容器的内容，例如查找、技术、遍历、寻找极值
  3. 迭代器：扮演了容器和算法之间的胶合剂
    1. 提供方法，能够依序访问某个容器所含的各个元素，而又无需暴露容器内部的表现方式。
    2. 每个容器都有自己的专属迭代器
    3. 分类：
种类 功能 支持运算
输入 只读访问 ++，==，!=
输出 只写访问 ++
前向 读写操作，并能向前推进迭代器 ++，==，!=
双向 读写操作，并能向前后推进迭代器 ++,–
随机访问 读写操作，并能以跳跃方式访问任意数据，功能最强 ++,–,[n],-n,<，<=,>,>=
  1. 仿函数：行为类似函数，可作为算法的某种策略
  2. 空间配置器：负责空间的配置与管理
  3. 配接器（适配器）：一种用来修饰容器或者仿函数或迭代器的东西

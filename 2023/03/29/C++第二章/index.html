一、继承

1. 基本使用

  1. 语法：class 子类 ：继承方式 父类
  2. 子类也称为派生类，父类也称为基类
  3. 派生类中成员包括：
    1. 一类是从基类继承的（共性）
    2. 一类是自己增加的成员（个性）
  4. 优点：减少重复代码

2. 继承方式

  1. 三种继承方式：
    1. 公共继承
    2. 保护继承
    3. 私有继承
  2. 总结
    1. 若父类成员属性是私有的，则无论子类使用何种方式继承，均不可访问
    2. （个人理解：）“保护>公共”，父类属性权限和继承方式取等级高的
      1. 当子类为公共继承时，除父类的私有属性外，其他继承的属性与父类权限一致
      2. 当子类为保护继承时，除父类的私有属性外，其他继承的属性均为保护权限
      3. 当子类为私有继承时，除父类的私有属性外，其他继承的属性均为私有权限

  1. 访问子类同名成员->直接访问即可
  2. 访问父类同名成员->需要加作用域

7. 多继承语法

  1. C++允许多重继承，一个类继承自多个父类
  2. 语法：class 子类：继承方式 父类1，继承方式 父类2...
  3. 多继承可能引发父类中有同名成员出现，需要加作用域区分
  4. 因此，实际开发中不建议使用多继承

8. 菱形继承

  1. 概念：

    1. 两个派生类B1,B2继承自同一个基类A
    2. 又有某个类C同时继承两个派生类B1,B2
    3. 菱形继承也称为钻石继承

  2. 存在问题：

    1. B1和B2继承了A的数据，当C使用数据时，会产生二义性
    2. C继承A的数据了两份，实际仅需要一份即可 -> 资源浪费

  3. 解决方法：虚继承

  4. 虚基类时仅有基类A存储了原始数据，而继承的两个派生类B1,B2存储的是vbptr(虚拟基类指针)，指向_k1,数据仅有一份

9. 类型转换

1. 隐式转换

  1. 在不需要明确指示的情况下，编译器会自动执行类型转换。

2. 显式转换

  • 强制类型转换
  • 四种类型转换操作符：
  1. static_cast<>
    1. 用于基本类型之间的转换、类层次结构中向上和向下转换、以及void*指针的转换
    2. 安全的向上转换（从派生类到基类）；向下转换（基类到派生类）但不安全，要求转换合理
    3. 经过静态类型检查的转换
  2. dynamic_cast<>
    1. 用于在类层次结构中进行安全的向下转换（只能用于多态类型，即含有虚函数的类）。
    2. 具有继承关系的类才能转
  3. const_cast<>
    1. 用于移除或添加对象的常量性，但不能改变底层的常量性（即指向真正的常量的指针不能去掉const）
  4. reinterpret_cast<>
    1. 用于位级别的转换，可以将指针或整数转换为任意类型的指针或整数，但这种转换是最低级的，可能导致不可预知的结果。
    2. 任意指针的地址级转换

二、多态与虚函数

1. 基本概念

1. 分类

  1. 静态多态：函数重载和运算符重载，复用函数名
  2. 动态多态：派生类和虚函数实现运行时的多态

2. 区别

  1. 静态多态的函数地址早绑定 - 编译阶段确定函数地址
  2. 动态多态的函数地址晚绑定 - 运行阶段确定函数地址

3. 动态多态满足条件

  1. 具有继承关系
  2. 子类重写父类函数（函数返回值类型，名称和参数列表完全相同）

4. 动态多态的使用

  • 父类的指针或者引用 指向子类对象

2. 底层原理

  1. 在C++中，向对象发送信息，就是调用对象上的方法
  2. 多态意味着调用成员函数时，会根据调用函数的对象类型来执行不同的函数过程
  3. 基类定义一个函数，子类对父类函数重写是重新并覆盖父类的实现
    1. 上例中Animal类的内部结构：vfptr(虚函数指针)指向vftable(记录虚函数地址)->&Animal::speak()
    2. 如果Cat类中未重写，则直接继承&Animal::speak()
    3. Cat类的内部结构：vfptr(虚函数指针)指向vftable(记录虚函数地址)->&Cat::speak()（覆盖掉了）

3. 多态的优点

  1. 代码组织结构清晰->快速定位某一具体功能
  2. 可读性强
  3. 利于前后期的拓展以及维护
    1. 真实开发中提倡开闭原则：对拓展进行开放，对修改进行关闭

4. 纯虚函数和抽象类

  1. 在多态中，通常父类中的虚函数是毫无意义的，主要都是调用子类重写的内容，因此可以将虚函数改为纯虚函数
  2. 纯虚函数语法：virtual 返回值类型 函数名（参数列表）= 0
  3. 当类中有了纯虚函数，这个类也被称为抽象类
  4. 抽象类特点：
    1. 无法实例化对象
    2. 子类必须重写抽象类中的纯虚函数，否则也属于抽象类

5. 虚析构和纯虚析构

  1. 问题：多态使用时，如果子类中有属性开辟到堆区，那么父类指针在释放时无法调用到子类的析构代码
  2. 解决：将父类中的析构函数改为虚析构或者纯虚析构
  3. 虚析构和纯虚析构共性:
    1. 可以解决父类指针释放子类对象
    2. 都需要有具体的函数实现
  4. 区别：
    1. 如果是纯虚析构，该类属于抽象类，无法实例化对象
  5. 虚析构语法：virtual ~类名(){};
  6. 纯虚析构语法：virtual ~类名() = 0;;类外实现:类名::类名(){}
  7. 如果子类中无堆区数据，可以不写这两者

三、重载

1. 函数重载

  1. 编译器根据参数类型产生代码，调用适当的函数
  2. 函数重载条件：
    1. 同一作用域下
    2. 函数名称相同
    3. 函数参数类型不同或者个数不同或者顺序不同
  3. 函数返回值不可以作为函数重载条件
  4. 注意事项
    1. 引用作为重载条件,const可以作为重载条件
    2. 函数重载碰到默认参数

2. 运算符重载

  1. 概念：对已有的运算符重新进行定义，赋予其另一种功能，以适应不同的数据类型
  2. 运算符是一种特殊名称的函数
  3. 基本语法：返回值类型 operator 运算符名称 (参数列表) { 对运算符的重载处理 }

0. 运算符重载规则

  1. 不允许定义新的运算符，只能对C++已有运算符进行重载
  2. 5个运算符不能重载
    1. .：成员访问运算符
    2. .*：成员指针访问运算符
    3. ::：域运算符
    4. sizeof：长度运算符
    5. ?:：条件运算符
  3. 重载不能改变运算符运算对象的个数
  4. 重载不能改变运算符的优先级和结合性
  5. 重载运算符的函数不能有默认参数，否则就改变了运算符参数的个数
  6. 重载的运算符必须和用户自定义类型的对象一起使用
  7. 使用语义相近的运算符来就行重载,不要滥用运算符重载，如把+号改为减运算
  8. 运算符重载函数可以是类的成员函数或者全局函数（可能为类的友元函数）

1. 加号运算符重载（双目运算符）

  1. 作用：实现两个自定义数据类型相加的运算
  2. 注意：
    1. 对于内置的数据类型的表达式的运算符不可改变，如1+1

2. 左移<<运算符重载（双目）

  1. 作用：输出自定义的数据类型

3. 递增++运算符重载（单目）

  1. 作用：通过重载递增运算符，实现自己的整型数据
  2. 前置递增返回引用，后置递增返回值

4. 赋值=运算符重载（双目）

  1. C++编译器至少给一个类添加4个函数：
    1. 默认构造函数（无参，空实现）
    2. 默认析构函数（无参，空实现）
    3. 默认拷贝构造函数，对属性进行值拷贝
    4. 赋值运算符operator=,对属性进行值拷贝
  2. 如果类中有属性指向堆区，做赋值操作时会出现深浅拷贝的问题

5. 关系运算符重载（双目）

  1. 作用：重载关系运算符，可以让两个自定义类型对象进行对比操作

6. 函数调用运算符重载

  1. 函数调用运算符()也可以重载
  2. 由于重载后使用的方法非常像函数的调用，因此称为仿函数
  3. 仿函数没有固定的写法，非常灵活
  4. 匿名函数对象

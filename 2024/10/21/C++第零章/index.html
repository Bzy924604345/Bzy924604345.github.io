一、基础知识

1. C++特点

  1. “完全”兼容C
  2. 编译、静态、大小写敏感、多编程范式

2. C++组成

  1. 核心语言
  2. C++标准库

3. C++程序生成过程：

  1. 源文件source（*.h,*.cpp）
  2. 预处理器preprocessor -> 编译器complier
  3. 生成目标文件（*.obj）
  4. 将目标文件和库文件（*.lib）一同传给链接器Linker
  5. 得到编译结果（*.lib,*dll,*exe）

二、指针

1. const修饰指针

1. 常量指针：const修饰指针

  1. 语法：const int* p = &a;
  2. 特点：指针的指向可以修改，但是指针指向的值不可以修改
  3. 记忆：const修饰int*，所以“指针”p可以改变值即改变指向，但不能修改指向值

2. 指针常量：const修饰常量

  1. 语法：int* const p = &a;
  2. 特点：指针指向的值不可以修改，但是指针的指向可以修改
  3. 记忆：const修饰的是p，因此“常量”p不可以再改变值即修改指向

3. const即修饰指针也修饰常量

  1. 语法：const int* const p = &a;
  2. 特点：指针的指向和指向的值都不可以修改

三、内存四区

  1. 程序编译后，生成exe可执行程序，未执行程序前分为两个区域：

    1. 代码区：存放函数体的二进制代码，由操作系统进行管理
      1. 存放CPU执行的机器指令
      2. 代码是共享的，目的是对于频繁被执行的程序，只要存储一份即可
      3. 代码是只读的，目的是防止程序意外修改指令
    2. 全局区
      1. 存储全局变量和静态变量
      2. 存储常量
      3. 该区域数据在程序结束后由操作系统释放

  2. 执行程序后：

    1. 栈区
      1. 存放函数的参数值，局部变量等
      2. 不要返回局部变量地址
      3. 栈区数据由编译器自动分配与释放
    2. 堆区
      1. 由程序员分配和释放，若不手动释放，会在程序结束时自动回收
      2. 使用new操作符在堆区开辟内存int* p =new int(10);double* arr = new double[10];
      3. 使用delete操作符手动释放delete p;delete[] arr;

四、引用

1. 基本使用

  1. 作用与本质：给变量取别名
  2. 语法：数据类型 &别名 = 原名;
  3. 别名和原名访问同一个内存

2. 注意事项

  1. 引用必须要初始化，错误：int &b;
  2. 一旦初始化后，不可以再修改引用，但可以赋值

3. 引用做函数参数

  1. 作用：函数传参时，可以利用引用让形参修饰实参
  2. 优点：可以简化指针修改实参
  3. 语法：

4. 引用做函数返回值

  1. 不能返回超出作用域的变量，如不要返回局部变量的引用
  2. 函数的调用可以作为左值

5. 引用的本质

引用的本质是在C++内部实现的是一个指针常量

int& ref = a;ref = 20;->int* const ref = &a;*ref = 20;

6. 常量引用

  1. 作用：常量引用主要用来在函数形参列表中用const修饰形参，防止误操作
  2. 其他示例：cosnt int& ref = 10;->int temp = 10;cosnt int& ref =temp;加入后变为只读，不可以修改值：ref = 20;

五、关于引用和指针的&,*符号的区分

  1. *
    1. 定义指针：int* p =&a;
    2. 解引用：*p->得到指针p指向的值
  2. &
    1. 定义引用：int &age = a;
    2. 取地址：(int)&a->以十进制输出a的内存地址

七、命名空间

  1. C++标准库的命名空间是std
  2. 各种第三方库都使用了命名空间
  3. 自定义命名空间

八、标准C++流库

1. <iostream>(位于std命名空间)

  1. cin:标准输入
  2. cout:标准输出
  3. clog：标准日志
  4. cerrL:标准错误

2. 格式控制

3. 流与文件

  1. 文件类型
    1. 文本文件：以ASCII码的形式存储
    2. 二进制文件：以二进制形式存储
  2. <ofstream>:该数据类型表示输出文件流，用于创建文件并向文件写入信息。
  3. <ifstream>:该数据类型表示输入文件流，用于从文件读取信息。
  4. <fstream>:该数据类型通常表示文件流，且同时具有 ofstream 和 ifstream 两种功能，这意味着它可以创建文件，向文件写入信息，从文件读取信息。

4. 文件流读写方法

  1. open：void open(const char* filename,int mode,int access);
  2. write：write ( char * buffer, streamsize size);
  3. read：read ( char * buffer, streamsize size );
  4. close

5. 文本文件-写文件步骤

  1. 包含头文件：#include<fstream>
  2. 创建流对象：ofstream ofs;
  3. 打开文件：ofs.open("文件路径",打开方式);
    1. ios::in:为读文件而打开文件
    2. ios::out:为写文件而打开文件
    3. ios::ate:初始位置：文件尾
    4. ios::app:追加方式写文件
    5. ios::trunc:如果文件存在先删除，再创建
    6. ios::binary:二进制方式
    7. 注意：文件打开方式可以配合使用，利用|操作符
  4. 写数据：ofs<<"写入的数据";
  5. 关闭文件：ofs.close();

6. 文本文件-读文件步骤

  1. 包含头文件：#include<fstream>
  2. 创建流对象：ifstream ifs;
  3. 打开文件并判断文件是否打开成功：ifs.open("文件路径",打开方式);,ifs.is_open()
  4. 读数据：四种方式读取
  5. 关闭文件：ifs.close();

7. 二进制文件-写文件步骤

  1. 打开方式加ios::binary
  2. 写二进制文件主要利用write
  3. 函数原型：ostream& write(const char* buffer,int len);

8. 二进制文件-读文件步骤

  1. 函数原型：ostream& read(const char* buffer,int len);

九、字符串与string类

  1. <string>:一个更强大的，面向对象的字符串类
    1. string:单字节字符串
    2. wstring:双字节字符串
  2. 查找方法族
    1. find：从前往后查找子串或字符出现的位置。
    2. rfind：从后往前查找子串或字符出现的位置。
    3. find_first_of：从前往后查找何处出现另一个字符串中包含的字符。
    4. find_last_of：从后往前查找何处出现另一个字符串中包含的字符。
    5. find_first_not_of：从前往后查找何处出现另一个字符串中没有包含的字符。
    6. find_last_not_of：从后往前查找何处出现另一个字符串中没有包含的字符。

十、函数的默认参数与占位参数

1. 函数默认参数

  1. 语法：返回值类型 函数名 （参数=默认值）{}
  2. 从第一个有默认值的位置起，后面的参数都需要有默认值（从左往右）
  3. 如果函数声明有默认参数，则函数实现不能有默认参数

2. 函数占位参数

  1. 形参可使用占位参数，但调用函数时必须填补该位置
  2. “传入值但不使用”
  3. 语法：返回值类型 函数名 （数据类型）{}

南京市地铁信息查询系统-服务设计报告

1. 服务概述

  • 南京市地铁信息查询系统主要的服务包括地铁站数据和地铁线数据，两者均为GIS Web服务

2. 请求说明

  • 本系统直接采用WFS服务，并利用axios请求数据。通过函数”getWFSData()”输入图层名和CQL查询语句请求数据（本质仍是WFS服务）

3. 服务设计

1. 获取地铁站数据

  1. 基本信息

    信息 值
    接口编号 WFS001
    要素名称 nj_2022_station_gcj02_4326
    几何类型 点
    坐标系统 GCJ-02(EPSG：4326)
    要素说明 南京市2022年10月地铁站数据（含规划和在建）
    数据来源 /
    数据处理 getWFSData()
    数据格式 JSON
    服务地址 http://localhost:8090/geoserver/webgis_test/ows?

  2. 属性设计

    字段名称 中文描述 数据类型 备注
    name 地铁站名 字符串 数据库中对name字段末尾的‘(地铁站)’进行了删除
    adname 所属区 字符串
    address 经停地铁及是否为在建 字符串

  3. 示例

2. 获取地铁线数据

  1. 基本信息

    API 设计
    接口编号 WFS002
    要素名称 nj_2022_subway_4326_gcj02_l
    几何类型 线
    坐标系统 GCJ-02(EPSG：4326)
    要素说明 南京市2022年10月地铁线数据
    数据来源 经人工预处理
    数据处理 getWFSData()
    数据格式 JSON
    服务地址 http://localhost:8090/geoserver/webgis_test/ows?

  2. 属性设计

    字段名称 中文描述 数据类型 备注
    name 地铁线名 字符串
    sum_length 地铁线总长 浮点型数值

  3. 示例
